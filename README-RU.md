## О проекте и особенности

[Посетите наш сайт](https://bazzite.gg/) для понятного объяснения, что такое Bazzite. Этот README подробнее раскрывает проект.

[Bazzite](https://bazzite.gg/) - это пользовательский [Fedora Atomic](https://fedoraproject.org/atomic-desktops/) образ, собранный на базе [cloud native](https://universal-blue.org/#cloud-native) технологий и дающий лучший Linux-опыт для игр **на всех ваших устройствах, включая любимые портативные**.

Bazzite собирается на основе [ublue-os/main](https://github.com/ublue-os/main) с использованием технологий [Fedora](https://fedoraproject.org/), поэтому в него входят расширенная поддержка оборудования и встроенные драйверы. Кроме того, Bazzite добавляет следующие возможности:

- Использует [ядро bazzite](https://github.com/bazzite-org/kernel-bazzite) для поддержки HDR и расширенной совместимости с оборудованием, а также многих других патчей.
- HDR доступен в игровом режиме.
- NVK доступен в сборках без Nvidia.
- Полная поддержка аппаратного ускорения декодирования H264.
- Полная поддержка ROCm OpenCL/HIP от AMD.
- Драйвер [xone](https://github.com/medusalix/xone) для контроллеров Xbox.
- Полная поддержка [DisplayLink](https://www.synaptics.com/products/displaylink-graphics).
- Включены темы KDE от Valve из SteamOS.
- [vkBasalt](https://github.com/DadSchoorse/vkBasalt), [MangoHud](https://github.com/flightlessmango/Mangohud) и [OBS VkCapture](https://github.com/nowrep/obs-vkcapture) установлены и доступны по умолчанию.
- [ROM Properties Page shell extension](https://github.com/GerbilSoft/rom-properties) включено.
- Полная поддержка [Winesync/Fastsync/NTsync](https://github.com/Frogging-Family/wine-tkg-git/issues/936).
- [Distrobox](https://github.com/89luca89/distrobox) предустановлен.
- Упрощенная установка Davinci Resolve с помощью [davincibox](https://github.com/zelikos/davincibox) (`ujust install-resolve`).
- Автоматическая служба `duperemove` для уменьшения места, занимаемого содержимым wine-префиксов.
- Поддержка HDMI CEC через [libCEC](https://libcec.pulse-eight.com/).
- По умолчанию используется [BBR TCP от Google](https://github.com/google/bbr).
- [Input Remapper](https://github.com/sezanzeb/input-remapper) предустановлен и включен. *(Доступен, но отключен по умолчанию в варианте Deck; можно включить командой `ujust restore-input-remapper`)*
- [Bazzite Portal](https://github.com/ublue-os/yafti-gtk) позволяет легко устанавливать множество приложений и настроек, включая [LACT](https://github.com/ilya-zlobintsev/LACT) и IDE через Brew. В нем также есть удобные кнопки для обновления, перехода на другой образ и даже сброса системного образа к настройкам по умолчанию.
- [Waydroid](https://waydro.id/) предустановлен для запуска Android-приложений. Настройка описана в [кратком руководстве](https://docs.bazzite.gg/Installing_and_Managing_Software/Waydroid_Setup_Guide/).
- Управляйте приложениями с помощью [Flatseal](https://github.com/tchx84/Flatseal), [Warehouse](https://github.com/flattool/warehouse) и [Gear Lever](https://github.com/mijorus/gearlever).
- [OpenRGB](https://gitlab.com/CalcProgrammer1/OpenRGB) с драйверами i2c-piix4 и i2c-nct6775 для управления RGB на некоторых материнских платах.
- [OpenRazer](https://openrazer.github.io) встроен, выберите OpenRazer в Bazzite Portal или выполните `ujust install-openrazer` в терминале, чтобы начать использовать его.
- Правила udev для [OpenTabletDriver](https://opentabletdriver.net/) встроены, а полный набор ПО можно установить через Bazzite Portal или командой `ujust install-opentabletdriver` в терминале.
- Поддержка клавиатур [Wooting](https://wooting.io/) из коробки.
- Встроенная поддержка AMD GPU Southern Islands *(HD 7000)* и Sea Islands *(HD 8000)* через драйвер `amdgpu`.
- [Webapp Manager](https://github.com/linuxmint/webapp-manager) доступен для создания приложений из веб-сайтов в разных браузерах, включая Firefox.

### Десктопная версия

Основной вариант `bazzite` подходит для настольных компьютеров.

- Автоматические обновления ОС, Flatpak и многого другого, которые обеспечивают [uupd](https://github.com/ublue-os/uupd) и [topgrade](https://github.com/topgrade-rs/topgrade).

> [!IMPORTANT]
> **ISO-образы можно скачать с нашего [сайта](https://download.bazzite.gg), а полезное руководство по установке доступно [здесь](https://docs.bazzite.gg/General/Installation_Guide/).**

Чтобы перейти с существующего Fedora Atomic на этот образ, если вам нужны **драйверы GPU с открытым кодом**:
(Обратите внимание: открытый вариант Mesa для GPU NVIDIA, NVK, по-прежнему может содержать ошибки. Если у вас есть проблемы, связанные с NVK, [сообщайте о них в Mesa](https://docs.mesa3d.org/bugs.html), а не в Ublue/Bazzite.)

```bash
rpm-ostree rebase ostree-unverified-registry:ghcr.io/ublue-os/bazzite:stable
```

или для устройств с GPU Nvidia, которым нужны **проприетарные драйверы NVIDIA**:

```bash
rpm-ostree rebase ostree-unverified-registry:ghcr.io/ublue-os/bazzite-nvidia:stable
```

**Для пользователей с включенным Secure Boot:** перед переходом следуйте [документации по Secure Boot](#secure-boot).

### Steam Deck/Домашние кинотеатры (HTPC)

Вариант, предназначенный для использования как альтернатива SteamOS на Steam Deck и для консольного опыта на HTPC, доступен как `bazzite-deck`:

- Загружается сразу в игровой режим, как в SteamOS.
- **Автоматический `duperemove` значительно уменьшает размер compatdata.**
- **Последняя версия Mesa создает меньшие кэши шейдеров и не требует их для предотвращения фризов.**
- **Можно загрузиться даже если диск заполнен.**
- **Поддержка всех языков, которые доступны в Fedora upstream.**
- **Использует Wayland на рабочем столе с [поддержкой Steam input](https://github.com/Supreeeme/extest).**
- Включен [HHD](https://github.com/hhd-dev/hhd) для расширенной поддержки ввода на не-Valve портативных устройствах.
- Включает перенесенные версии большинства пакетов SteamOS, включая драйверы, средства обновления прошивок и контроллеры вентиляторов [из репозитория evlaV](https://gitlab.com/evlaV).
- Исправленная Mesa для корректного управления частотой кадров из Gamescope.
- Поставляется с патчами из [SteamOS BTRFS](https://gitlab.com/popsulfr/steamos-btrfs) для полной поддержки BTRFS на SD-карте по умолчанию.
- В комплекте перенесенная версия [SDGyroDSU](https://github.com/kmicki/SteamDeckGyroDSU), включенная по умолчанию.
- Есть возможность установить [Decky Loader](https://github.com/SteamDeckHomebrew/decky-loader), [EmuDeck](https://www.emudeck.com/), [RetroDECK](https://retrodeck.net/) и [ProtonUp-Qt](https://davidotek.github.io/protonup-qt/), а также множество других полезных пакетов при установке.
- Собственная система обновлений позволяет обновлять ОС, Flatpak и многое другое прямо из интерфейса игрового режима с помощью [uupd](https://github.com/ublue-os/uupd) и [topgrade](https://github.com/topgrade-rs/topgrade).
- Встроена поддержка двойной загрузки Windows благодаря тому, что установка GRUB от Fedora остается нетронутой.
- Обновление сломало что-то? Легко откатитесь к предыдущей версии Bazzite благодаря функции отката `rpm-ostree`. Можно даже выбрать предыдущие образы при загрузке.
- Steam и Lutris предустановлены в образе как добавленные пакеты.
- Используется ZRAM *(4GB)* с алгоритмом сжатия LZ4 по умолчанию.
- [LAVD](https://crates.io/crates/scx_lavd) и [BORE](https://github.com/firelzrd/bore-scheduler) как планировщики CPU для плавного и отзывчивого геймплея.
- Планировщик I/O Kyber, чтобы избежать сбоев ввода-вывода при установке игр или во время фонового процесса `duperemove`.
- Применяются параметры ядра SteamOS.
- Включены профили калибровки дисплея для матовых и зеркальных экранов Steam Deck.
- Функции для опытных пользователей, отключенные по умолчанию, включая:
  - Сервис для безопасного понижения напряжения Steam Deck и ноутбуков AMD Framework через [RyzenAdj](https://github.com/FlyGoat/RyzenAdj) и [Ryzen SMU](https://gitlab.com/leogx9r/ryzen_smu), см. `ryzenadj.service` и `/etc/default/ryzenadj`.
  - Поддержка разгона дисплея. Например, добавьте `CUSTOM_REFRESH_RATES=30-68` в `/etc/environment`. Минимальная и максимальная частоты обновления зависят от устройства.
  - У вас Steam Deck с модом на 32GB RAM? Наслаждайтесь автоматически применяемым удвоенным максимумом VRAM. *(Можете ли вы поделиться своими навыками пайки?)*
- Аппаратные сервисы Steam Deck можно отключить командой `ujust disable-bios-updates` и `ujust disable-firmware-updates` в терминале. На не-Deck оборудовании и на Deck с экранами DeckHD или модами 32GB RAM они отключены автоматически.
- Дополнительную информацию можно найти [здесь](https://docs.bazzite.gg/Handheld_and_HTPC_edition/Steam_Gaming_Mode/).

> [!IMPORTANT]
> **ISO-образы можно скачать с нашего [сайта](https://download.bazzite.gg), а полезное руководство по установке доступно [здесь](https://docs.bazzite.gg/General/Installation_Guide/).**

Чтобы перейти с существующего Fedora Atomic на этот образ:

```bash
rpm-ostree rebase ostree-unverified-registry:ghcr.io/ublue-os/bazzite-deck:stable
```

#### Альтернативные портативные устройства

Пожалуйста, обратитесь к нашему [Handheld Wiki](https://docs.bazzite.gg/Handheld_and_HTPC_edition/Handheld_Wiki/) за нужными изменениями настроек и плагинами Decky Loader для Steam Gaming Mode на вашем конкретном портативном устройстве.

**Обязательно также прочитайте [документацию HHD](https://github.com/hhd-dev/hhd#after-install). Для некоторых устройств нужны отдельные изменения настроек.**

Мы также поставляем команды `ujust` для установки разных тем [CSS Loader](https://docs.deckthemes.com/CSSLoader/Install/#linux-or-steam-deck), которых нет в магазине CSS Loader. Если они установлены, они будут автоматически обновляться вместе с Bazzite.

```bash
# Установить тему Handheld Controller Theme (https://github.com/victor-borges/handheld-controller-glyphs)
ujust install-hhd-controller-glyph-theme
```

### GNOME

Сборки с GNOME доступны и в desktop, и в deck-вариантах. Эти сборки идут со следующими дополнительными возможностями:

- [Поддержка переменной частоты обновления и дробного масштабирования в Wayland](https://gitlab.gnome.org/GNOME/mutter/-/merge_requests/1154).
- Собственное меню в верхней панели для возврата в игровой режим, запуска Steam и открытия ряда полезных утилит.
- [GSConnect](https://extensions.gnome.org/extension/1319/gsconnect/) предустановлен и готов к использованию.
- [Расширение Hanabi](https://github.com/jeffshee/gnome-ext-hanabi) включено, чтобы дать в KDE похожие возможности на замену Wallpaper Engine.
- Предустановлено множество дополнительных расширений, включая [важные исправления пользовательского опыта](https://www.youtube.com/watch?v=nbCg9_YgKgM).
- Автоматические обновления для темы [Firefox GNOME](https://github.com/rafaelmardojai/firefox-gnome-theme) и [Thunderbird GNOME](https://github.com/rafaelmardojai/thunderbird-gnome-theme). *(Если установлены)*

> [!IMPORTANT]
> **ISO-образы можно скачать с нашего [сайта](https://download.bazzite.gg), а полезное руководство по установке доступно [здесь](https://docs.bazzite.gg/General/Installation_Guide/).**

Чтобы перейти с существующего Fedora Atomic на этот образ:

```bash
rpm-ostree rebase ostree-unverified-registry:ghcr.io/ublue-os/bazzite-gnome:stable
```

Чтобы перейти на desktop environment с **проприетарными драйверами NVIDIA**:

```bash
rpm-ostree rebase ostree-unverified-registry:ghcr.io/ublue-os/bazzite-gnome-nvidia:stable
```

Чтобы перейти на **Steam Deck/HTPC**:

```bash
rpm-ostree rebase ostree-unverified-registry:ghcr.io/ublue-os/bazzite-deck-gnome:stable
```

#### Universal Blue

- Проприетарные драйверы Nvidia предустановлены. *(Только для образов Nvidia)*

#### Особенности Fedora Linux (Kinoite и Silverblue)

- Надежная и стабильная база.
- Системные пакеты остаются относительно актуальными.
- Можно добавлять пакеты Fedora в образ без потери при обновлениях.
- Безопасность, ориентированная на [SELinux](https://github.com/SELinuxProject/selinux), предустановленный и настроенный из коробки.
- Возможность перейти на другие образы Fedora Atomic, если потребуется, без потери пользовательских данных.
- Поддержка печати благодаря предустановленному [CUPS](https://www.cups.org/).

## Почему Bazzite?

Bazzite появился как проект, призванный решить некоторые проблемы SteamOS, прежде всего устаревшие пакеты (несмотря на базу Arch) и отсутствие функционального менеджера пакетов.

Хотя этот проект тоже основан на образах, вы можете устанавливать любой пакет Fedora прямо из командной строки. Эти пакеты сохраняются между обновлениями *(Так что можете смело поставить тот странный VPN, который целый час пытались заставить работать в SteamOS)*. Кроме того, Bazzite обновляется несколько раз в неделю пакетами из Fedora upstream, давая вам максимум производительности и свежих возможностей - и все это на стабильной базе.

Bazzite поставляется с самым свежим ядром Linux и SELinux, включенным по умолчанию, с полной поддержкой Secure Boot *(Запустите `ujust enroll-secure-boot-key` и введите пароль `universalblue`, если появится запрос на добавление нашего ключа)* и шифрования диска, что делает его разумным выбором для повседневной работы. *(Да, из Bazzite можно печатать)*

Ознакомьтесь с [FAQ](https://docs.bazzite.gg/General/FAQ/), чтобы узнать, чем Bazzite отличается от других Linux-систем.

## Проверка подлинности

Эти образы подписаны с помощью [cosign](https://docs.sigstore.dev/cosign/signing/overview/). Чтобы проверить подпись, скачайте ключ `cosign.pub` из этого репозитория и выполните команду:

```bash
cosign verify --key cosign.pub ghcr.io/ublue-os/bazzite
```

## Secure Boot

> [!WARNING]
> **Пользователи Steam Deck: на Steam Deck Secure Boot по умолчанию не включен и никакие ключи не зарегистрированы. Не включайте его, если не уверены, что делаете.**

Secure Boot поддерживается нашим собственным ключом. Публичный ключ можно найти в корне этого репозитория [здесь](https://github.com/ublue-os/bazzite/blob/main/secure_boot.der).
Если вы хотите зарегистрировать этот ключ до установки или перехода на другой образ, скачайте ключ и выполните:

```bash
sudo mokutil --timeout -1
sudo mokutil --import secure_boot.der
```

Для пользователей, которые уже используют образ Universal Blue, можно вместо этого выполнить `ujust enroll-secure-boot-key`.

Если система попросит пароль, используйте `universalblue`.

## Соберите свою версию

Bazzite полностью собирается через GitHub Actions. Создать собственную версию так же просто, как форкнуть этот репозиторий, добавить закрытый ключ для подписи и включить GitHub Actions в форке. Запуск workflow `Build Bazzite` создаст собственные образы для всех вариантов Bazzite.

Если вы хотите собирать только те варианты Bazzite, которыми пользуетесь сами, отредактируйте `.github/workflows/build.yml` и закомментируйте те варианты, которые не хотите собирать, в списке `matrix` для job `push-ghcr`.

Мы также поставляем конфигурацию для популярного [pull app](https://github.com/apps/pull), если хотите держать свой форк синхронизированным с upstream. Включите это приложение в своем репозитории, чтобы отслеживать изменения Bazzite и при этом вносить собственные правки.

### Как подписывать свои образы

1. Сначала [ознакомьтесь](https://docs.github.com/en/actions/security-guides/encrypted-secrets) с тем, как хранить секреты в GitHub.
2. [Сгенерируйте новую пару ключей](https://docs.sigstore.dev/cosign/key_management/signing_with_self-managed_keys/) с помощью Cosign (`cosign generate-key-pair`). У этой пары ключей не должно быть подписи.
3. Замените файл `cosign.pub` в публичном репозитории на сгенерированный вами - он понадобится вам и вашим пользователям для проверки подписей.
4. Добавьте приватный ключ, который хранится в виде текста в файле `cosign.key`, как Repository Secret на странице настроек форка, в меню `Settings -> Secrets and variables -> Actions`. Назовите секрет `SIGNING_SECRET`.
