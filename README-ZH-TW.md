## 關於 & 功能

[請看我們的網站](https://bazzite.gg/) 獲得 Bazzite 的新手入門解說。這個 readme 將深入探討所有細節。

[Bazzite](https://bazzite.gg/) 是一個透過 [原生雲端](https://universal-blue.org/#cloud-native) 技術建立的自訂 [Fedora Atomic](https://fedoraproject.org/atomic-desktops/) 映像檔，將最好的 Linux 遊戲體驗帶入**您所有的裝置 - 包含您最喜愛的掌上型裝置**。

Bazzite 採用 [Fedora](https://fedoraproject.org/) 技術並透過 [ublue-os/main](https://github.com/ublue-os/main) 與 [ublue-os/nvidia](https://github.com/ublue-os/nvidia) 打造而成，這意味著它包含了更廣泛的硬體支援與內建驅動程式。此外，Bazzite 還增加了以下功能：

- 以 [fsync 核心(kernal)](https://copr.fedorainfracloud.org/coprs/sentry/kernel-fsync/) 為靈感 - 使用 [bazzite 核心](https://github.com/bazzite-org/kernel-bazzite) 以實現 HDR 和廣泛的硬體支援，以及眾多其他包含的修補 (patch)。
- 遊戲模式 (Game mode) 下支援 HDR。
- 在非 Nvidia 版本上支援 NVK。
- 完整支援 H264 解碼的硬體加速編解碼器。
- 完整支援 AMD 的 ROCM OpenCL/HIP 執行環境 (run-times)。
- 適用於 Xbox 控制器的 [xone](https://github.com/medusalix/xone) 驅動程式。
- 完整支援 [DisplayLink](https://www.synaptics.com/products/displaylink-graphics)。
- 包含來自 SteamOS 的 Valve KDE 主題。
- 預設安裝並可使用 [LatencyFleX](https://github.com/ishitatsuyuki/LatencyFleX)、[vkBasalt](https://github.com/DadSchoorse/vkBasalt)、[MangoHud](https://github.com/flightlessmango/Mangohud) 與 [OBS VkCapture](https://github.com/nowrep/obs-vkcapture)。
- 包含 [ROM Properties Page shell extension](https://github.com/GerbilSoft/rom-properties)。
- 完整支援 [Winesync/Fastsync/NTsync](https://github.com/Frogging-Family/wine-tkg-git/issues/936)。
- 預先安裝 [Distrobox](https://github.com/89luca89/distrobox)。
- 透過 [davincibox](https://github.com/zelikos/davincibox) (`ujust install-resolve`) 簡化 Davinci Resolve 的安裝。
- 自動化 `duperemove` 服務，用於減少 wine prefix 內容所佔用的磁碟空間。
- 透過 [libCEC](https://libcec.pulse-eight.com/) 支援 HDMI CEC。
- 預設使用 [Google 的 BBR TCP 擁塞控制 (congestion control)](https://github.com/google/bbr)。
- 預裝並啟用 [Input Remapper](https://github.com/sezanzeb/input-remapper)。 <sub><sup>(在 Deck 版本中可用但預設停用，可透過 `ujust restore-input-remapper` 啟用)</sup></sub>
- [Bazzite Portal](https://github.com/ublue-os/yafti-gtk) 提供了一種簡單的方法來安裝眾多應用程式與調校，包括透過 Brew 安裝 [LACT](https://github.com/ilya-zlobintsev/LACT) 和 IDE。此外，它還提供了簡單的按鈕來更新、重定基底 (rebase)，甚至將系統映像檔重設為預設值。
- 預裝 [Waydroid](https://waydro.id/) 以執行 Android 應用程式。請透過此 [快速指南](https://docs.bazzite.gg/Installing_and_Managing_Software/Waydroid_Setup_Guide/) 進行設定。
- 使用 [Flatseal](https://github.com/tchx84/Flatseal)、[Warehouse](https://github.com/flattool/warehouse) 與 [Gear Lever](https://github.com/mijorus/gearlever) 管理應用程式。
- [OpenRGB](https://gitlab.com/CalcProgrammer1/OpenRGB) i2c-piix4 與 i2c-nct6775 驅動程式，用於控制特定主機板上的 RGB。
- 內建 [OpenRazer](https://openrazer.github.io) 驅動程式。在 Bazzite Portal 中選擇 OpenRazer 或在終端機中執行 `ujust install-openrazer` 即可開始使用。
- 內建 [OpenTabletDriver](https://opentabletdriver.net/) udev 規則，完整的軟體套件可透過 Bazzite Portal 安裝，或在終端機中執行 `ujust install-opentabletdriver` 安裝。
- 開箱即用支援 [Wooting](https://wooting.io/) 鍵盤。
- 在 `amdgpu` 驅動下內建支援 Southern Islands <sub><sup>(HD 7000)</sup></sub> 與 Sea Islands <sub><sup>(HD 8000)</sup></sub> AMD GPU。
- 在 Wayland 上可使用 [XwaylandVideoBridge](https://invent.kde.org/system/xwaylandvideobridge) 進行 Discord 螢幕分享。
- 可使用 [Webapp Manager](https://github.com/linuxmint/webapp-manager) 從各種瀏覽器 (包括 Firefox) 的網站建立應用程式。

### 桌上型電腦

通用的 `bazzite` 版本，適合桌上型電腦使用。

- 作業系統、Flatpak 等皆會自動更新 - 由 [ublue-update](https://github.com/ublue-os/ublue-update) 與 [topgrade](https://github.com/topgrade-rs/topgrade) 驅動。

> [!IMPORTANT]
> **ISO 映像檔可從我們的 [網站](https://download.bazzite.gg) 下載，並可在此找到實用的 [安裝指南](https://docs.bazzite.gg/General/Installation_Guide/)。**

若您想要使用 **開源 GPU 驅動程式**，可從現有的上游 Fedora Atomic 重定基底 (Rebase) 至此映像檔：
(請注意：Mesa 對於 NVIDIA GPU 的開源選項 NVK，在撰寫本文時仍容易出錯。若有任何關於 NVK 的問題，[請向 Mesa 提交報告]([url](https://docs.mesa3d.org/bugs.html))，而非 Ublue/Bazzite)

```bash
rpm-ostree rebase ostree-unverified-registry:ghcr.io/ublue-os/bazzite:stable
```

或者若您的裝置搭載 Nvidia GPU 並想要使用 **NVIDIA 專有驅動程式**：

```bash
rpm-ostree rebase ostree-unverified-registry:ghcr.io/ublue-os/bazzite-nvidia:stable
```

**針對已啟用安全啟動 (Secure Boot) 的使用者：** 在重定基底 (Rebase) 之前，請遵循我們的 [安全啟動文件](#安全啟動)。

### Steam Deck / 家庭式劇院個人電腦 (Home Theater PCs, HTPCs)
專為 Steam Deck 作為 SteamOS 替代品以及 HTPC 上類似主機體驗所設計的版本，名稱為 `bazzite-deck`：

- 依循 SteamOS 的行為，直接啟動進入遊戲模式 (Game mode)。
- **自動執行 `duperemove` 大幅縮減 compatdata 的大小。**
- **最新版本的 Mesa 建立較小的著色器快取 (shader caches)，且不需要它們即可防止卡頓。**
- **即使磁碟已滿也能正常啟動。**
- **支援上游 Fedora 所支援的所有語言。**
- **在桌面上使用 Wayland，並 [支援 Steam 輸入](https://github.com/Supreeeme/extest)。**
- 包含 [HHD](https://github.com/hhd-dev/hhd) 以在非 Valve 的掌上機上提供擴展的輸入支援。
- 包含多數 SteamOS 套件的移植版本，包括驅動程式、韌體更新程式和風扇控制器，皆來自 [evlaV 儲存庫](https://gitlab.com/evlaV)。
- 包含經過修補的 Mesa，以支援 Gamescope 的正確幀率控制。
- 預設包含來自 [SteamOS BTRFS](https://gitlab.com/popsulfr/steamos-btrfs) 的修補程式，以完整支援 SD 卡的 BTRFS 格式。
- 預裝並啟用 [SDGyroDSU](https://github.com/kmicki/SteamDeckGyroDSU) 的移植版本。
- 安裝時可選擇安裝 [Decky Loader](https://github.com/SteamDeckHomebrew/decky-loader)、[EmuDeck](https://www.emudeck.com/)、[RetroDECK](https://retrodeck.net/) 與 [ProtonUp-Qt](https://davidotek.github.io/protonup-qt/) 等眾多實用套件。
- 自訂的更新系統允許直接從遊戲模式介面更新作業系統、Flatpaks 等，由 [ublue-update](https://github.com/ublue-os/ublue-update) 與 [topgrade](https://github.com/topgrade-rs/topgrade) 驅動。
- 由於保留了 Fedora 的 GRUB 安裝，內建支援與 Windows 雙重開機 (Duel-boot)。
- 更新後出問題了嗎？透過 `rpm-ostree` 的 rollback 功能，可輕鬆還原至 Bazzite 的前一個版本。您甚至可以在開機時選擇先前的映像檔。
- Steam 與 Lutris 已作為分層套件 (layered packages) 預裝在映像檔中。
- Discord 的 [Discover Overlay](https://github.com/trigg/Discover) 已預裝，若安裝了 Discord，將自動在遊戲模式與桌面上啟動。[點此查看官方文件](https://trigg.github.io/Discover/bazzite)。
- 預設使用採用 LZ4 壓縮演算法的 ZRAM<sub><sup>(4GB)</sup></sub>。
- 使用 [LAVD](https://crates.io/crates/scx_lavd) 與 [BORE](https://github.com/firelzrd/bore-scheduler) CPU 排程器，提供流暢且靈敏的遊戲體驗。
- 使用 Kyber I/O 排程器以防止在安裝遊戲或背景執行 `duperemove` 時發生 I/O 資源匱乏 (starvation)。
- 套用 SteamOS 的 kernel 參數。
- 包含適用於霧面 (matte) 和反光 (reflective) Steam Deck 螢幕的色彩校正設定檔。
- 預設停用的進階使用者功能，包含：
    - 透過 [RyzenAdj](https://github.com/FlyGoat/RyzenAdj) 與 [Ryzen SMU](https://gitlab.com/leogx9r/ryzen_smu) 為 Steam Deck 及 AMD Framework 筆記型電腦提供低風險的降壓服務，請參閱 `ryzenadj.service` 與 `/etc/default/ryzenadj`。
    - 內建螢幕超頻支援。例如，新增 `CUSTOM_REFRESH_RATES=30-68` 至 `/etc/environment`。最小與最大更新率因掌上機而異！
    - 您有將 Steam Deck 改裝為 32GB RAM 嗎？享受雙倍的最大 VRAM 容量，將自動套用。<sup><sub>(可以分享一下您的焊接技巧嗎？)</sub></sup>
- Steam Deck 硬體專屬服務可透過在終端機執行 `ujust disable-bios-updates` 和 `ujust disable-firmware-updates` 來停用。這些服務在非 Deck 硬體，以及擁有 DeckHD 螢幕或 32GB RAM 改裝的 Deck 上會自動停用。
- 更多關於 Bazzite Steam Deck 映像檔的資訊可以在 [這裡](https://docs.bazzite.gg/Handheld_and_HTPC_edition/Steam_Gaming_Mode/) 找到。

> [!IMPORTANT]
> **ISO 映像檔可從我們的 [網站](https://download.bazzite.gg) 下載，並可在此找到實用的 [安裝指南](https://docs.bazzite.gg/General/Installation_Guide/)。**

從現有的上游 Fedora Atomic 重定基底 (Rebase) 至此映像檔：

```bash
rpm-ostree rebase ostree-unverified-registry:ghcr.io/ublue-os/bazzite-deck:stable
```

#### 其他的掌上型裝置

請參考我們的 [掌上型裝置 Wiki](https://docs.bazzite.gg/Handheld_and_HTPC_edition/Handheld_Wiki/)，了解您特定掌上型裝置在 Steam 遊戲模式下所需的設定變更與 Decky Loader 外掛程式。

**請務必閱讀 [hhd 文件](https://github.com/hhd-dev/hhd#after-install)，部分掌上機需要特定的設定變更/調整才能正常運作。**

我們也提供了 `ujust` 指令來安裝各種無法在 CSS Loader 商店中找到的 [CSS Loader](https://docs.deckthemes.com/CSSLoader/Install/#linux-or-steam-deck) 主題。如果安裝了這些主題，它們將隨 Bazzite 自動更新。
```bash
# 安裝掌上機控制器主題 (https://github.com/victor-borges/handheld-controller-glyphs)
ujust install-hhd-controller-glyph-theme
```

### GNOME

GNOME 桌面環境的版本同時提供桌面版 (desktop) 與掌機版 (deck)。這些版本具有以下額外功能：

- [在 Wayland 下支援可變更新率與分數縮放](https://gitlab.gnome.org/GNOME/mutter/-/merge_requests/1154)。
- 頂端列 (top bar) 的自訂選單，用於返回遊戲模式、啟動 Steam 與開啟多個實用的工具。
- 預裝並可立即使用的 [GSConnect](https://extensions.gnome.org/extension/1319/gsconnect/)。
- 包含 [Hanabi 擴充功能](https://github.com/jeffshee/gnome-ext-hanabi)，提供類似 KDE 中 Wallpaper Engine 的功能。
- 預裝許多可選的擴充功能，包括 [重要的使用者體驗修復](https://www.youtube.com/watch?v=nbCg9_YgKgM)。
- 自動更新 [Firefox GNOME 主題](https://github.com/rafaelmardojai/firefox-gnome-theme) 與 [Thunderbird GNOME 主題](https://github.com/rafaelmardojai/thunderbird-gnome-theme)。<sup><sub>(如果已安裝)</sub></sup>

> [!IMPORTANT]
> **ISO 映像檔可從我們的 [網站](https://download.bazzite.gg) 下載，並可在此找到實用的 [安裝指南](https://docs.bazzite.gg/General/Installation_Guide/)。**

從現有的上游 Fedora Atomic 重定基底 (Rebase) 至此映像檔：

```bash
rpm-ostree rebase ostree-unverified-registry:ghcr.io/ublue-os/bazzite-gnome:stable
```

將現有的 ostree 系統重定基底至帶有 **NVIDIA 專有驅動程式** 的桌面環境版本：

```bash
rpm-ostree rebase ostree-unverified-registry:ghcr.io/ublue-os/bazzite-gnome-nvidia:stable
```

將現有的 ostree 系統重定基底至 **Steam Deck/HTPC** 版本：

```bash
rpm-ostree rebase ostree-unverified-registry:ghcr.io/ublue-os/bazzite-deck-gnome:stable
```

**針對已啟用安全啟動 (Secure Boot) 的使用者：** 在重定基底 (Rebase) 之前，請遵循我們的 [安全啟動文件](#安全啟動)。

### 上游系統特色

#### Universal Blue

- 預裝專有 Nvidia 驅動程式。<sub><sup>(僅限 Nvidia 映像檔)</sup></sub>
- 預設啟用 Flathub。
- 便捷的 [`ujust`](https://github.com/casey/just) 指令。
- 開箱即用的多媒體編解碼器。
- 可從過去 90 天內的任何版本回滾 (Rollback) Bazzite。

#### 來自 Fedora Linux 的特色 (Kinoite & Silverblue)

- 堅若磐石且穩定的基礎。
- 系統套件保持相對較新。
- 可在映像檔上分層 (layer) 安裝 Fedora 套件，且不會在更新間遺失。
- 專注於安全性，預裝並預設定好 [SELinux](https://github.com/SELinuxProject/selinux)。
- 如果需要，能夠重定基底 (rebase) 至不同的 Fedora Atomic 映像檔，而不會遺失使用者資料。
- 預裝 [CUPS](https://www.cups.org/) 以支援列印功能。

## 目的

Bazzite 最初是一個旨在解決 SteamOS 一些問題的專案，主要是過舊的軟體包（儘管基於 Arch）以及缺乏功能完善的軟體包管理器。

儘管此專案也是基於映像檔的系統，但您可以直接從命令列安裝任何 Fedora 軟體包。這些軟體包將在更新後保留 <sub><sup>(所以放心安裝那些您花了許多時間試圖在 SteamOS 中運作的冷門 VPN 軟體吧)</sup></sub>。此外，Bazzite 每週多次從上游 Fedora 更新軟體包，在穩定的基礎上為您提供最佳效能和最新功能。

Bazzite 預設啟用最新的 Linux 核心和 SELinux，並完整支援安全啟動 <sub><sup>(若提示輸入密碼以註冊我們的金鑰，請執行 `ujust enroll-secure-boot-key` 並輸入密碼 `universalblue`)</sup></sub> 和磁碟加密，使其成為一般電腦使用的明智解決方案。<sup><sub>(是的，您可以從 Bazzite 列印)</sub></sup>

閱讀 [常見問題 (FAQ)](https://docs.bazzite.gg/General/FAQ/) 了解 Bazzite 與其他基於 Linux 的作業系統有何不同。

## 驗證

這些映像檔均使用 sigstore 的 [cosign](https://docs.sigstore.dev/cosign/key_management/overview/) 進行簽署。您可以從此儲存庫下載 `cosign.pub` 金鑰並執行以下指令來驗證簽章：

```bash
cosign verify --key cosign.pub ghcr.io/ublue-os/bazzite
```

## 安全啟動

> [!WARNING]
> **Steam Deck 使用者：Steam Deck 預設並未啟用安全啟動 (Secure Boot)，也未寫入任何金鑰。除非您非常清楚自己在做什麼，否則請勿啟用此功能。**

我們的自訂金鑰支援安全啟動。公開金鑰 (pub key) 可以在此儲存庫的根目錄中找到 [這裡](https://github.com/ublue-os/bazzite/blob/main/secure_boot.der)。
若您想在安裝或重定基底 (rebase) 之前註冊此金鑰，請下載金鑰並執行以下指令：

```bash
sudo mokutil --timeout -1
sudo mokutil --import secure_boot.der
```

對於已經在使用 Universal Blue 映像檔的使用者，您可以改為執行 `ujust enroll-secure-boot-key`。

如果被要求輸入密碼，請使用 `universalblue`。

## 自行編譯

Bazzite 完全在 GitHub 上建置，建立您自己的自訂版本非常簡單，只需 Fork 此儲存庫、新增私有簽章金鑰並啟用 GitHub Actions 即可。

請先 [熟悉](https://docs.github.com/en/actions/security-guides/encrypted-secrets) 如何在 GitHub 中保存 Secret (機密)。您需要使用 cosign [產生一組新的金鑰對](https://docs.sigstore.dev/cosign/signing/overview/)。公開金鑰 (Public key) 可以放在您的公開 Repo 中 <sub><sup>(您的使用者需要它來檢查簽章)</sup></sub>，您可以將私有金鑰 (Private key) 貼在 `Settings -> Secrets -> Actions` 中，名稱設為 `SIGNING_SECRET`。

如果您希望您的分支與上游保持同步，我們也提供了流行的 [pull app](https://github.com/apps/pull) 設定檔。在您的 Repo 上啟用此應用程式，即可在進行您自己的修改的同時，追蹤 Bazzite 的變更。
